# NexTarget-server v0.2.0 — 2026-07-09

Release des sprints S1 (Sécurité), S2 (Demo-ready) et S3 (Robustesse).

## Sécurité (S1)
- **NT-065** — CORS restreint par environnement : `*` en dev, aucune origine en
  production sauf `CORS_ALLOW_ORIGINS` explicite (l'app mobile n'est pas concernée).
- **NT-066** — Nonce OIDC Google vérifié au callback (anti-rejeu d'id_token).

## Coach IA (S2)
- **NT-032** — Persona `coach_cool` (ton encourageant) en plus de `coach_neutre` ;
  sélection via `prompt_variant`, défaut inchangé.

## Robustesse (S3)
- **NT-055** — CI GitHub Actions : pytest + couverture (Python 3.11).
- **NT-054** — Flows OAuth Google/Facebook entièrement testés avec providers mockés.
- **NT-048** — Refresh tokens : `/auth/token/refresh` (rotation usage unique,
  rejeu ⇒ révocation de famille), `/auth/token/revoke` (logout). Champs additifs
  sur `/auth/token/exchange` — **contrat client existant inchangé**.
- **NT-053** — Logs JSON structurés + corrélation `X-Request-ID`.

## Compatibilité & déploiement
- **Aucune migration de schéma destructive** : la table `refresh_token` est créée
  automatiquement au démarrage (`init_db`). SQLite conservé.
- **Aucun breaking change d'API.**
- Variables d'env : aucune nouvelle obligatoire. Optionnelles : `LOG_LEVEL`
  (défaut INFO), `REFRESH_TOKEN_EXP_DAYS` (défaut 30), `CORS_ALLOW_ORIGINS`
  (défaut : aucune origine en prod).
- ⚠️ Prérequis de mise en prod : **rotation de la clé `MISTRAL_API_KEY`**
  (l'ancienne clé a été exposée côté client / historique git de l'app).

Tests : 64 verts + 1 skip conditionnel (65 en CI). Détail : `CHANGELOG.md`.
